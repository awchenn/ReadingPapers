# Architecture decision records

ADRs preserve the context behind foundational choices. A decision is not silently rewritten after implementation begins; create a new ADR that supersedes it.

| ADR | Decision | Status |
| --- | --- | --- |
| [0001](0001-local-first-client-only-pwa.md) | Local-first, client-only PWA | Accepted |
| [0002](0002-one-active-paper.md) | One active paper in the MVP | Accepted |
| [0003](0003-connected-bullet-outlines.md) | Connected bullet outlines instead of named threads | Accepted |
| [0004](0004-pdf-coordinate-anchors.md) | PDF-space rectangles as durable anchors | Accepted |

## ADR template

New records should contain a title, status, date, context, decision, consequences, and links to affected design documents.
